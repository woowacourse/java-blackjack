# java-blackjack

블랙잭 미션 저장소

---

### 블랙잭

플레이어와 딜러가 카드를 나누어 받아, 그 합이 21 또는 21에 가까운 숫자를 얻어 승부를 겨루는 게임

**<용어>**

* `블랙잭(Blackjack)` - 처음 두 장의 카드의 합이 21일 경우, 베팅 금액의 1.5배 Pay
* `버스트(Bust)` - 카드 합이 21을 초과하는 경우, 플레이어의 경우 Bust되면 베팅 금액을 잃게 되고 딜러의 경우에는 살아남은 플레이어에게 1:1 Pay
* `스탠드(Stand)` - 플레이어가 추가 카드를 원하지 않을 경우, 딜러의 경우 17 이상이 되면 무조건 Stand
* `힛(Hit)` - 플레이어가 추가 카드를 딜러에게 요청
* `푸시(Push)` - 딜러와 플레이어 모두 블랙잭이거나 점수가 같을 경우 무승부, 베팅 금액 반환

**<게임 승패 판단>**

1. 블랙잭 체크 - 딜러 및 플레이어 모두 블랙잭이면 무승부 / 한 명만 블랙잭이면 블랙잭 승리
2. 버스트 체크 - 딜러 및 플레이어 모두 버스트면 딜러 승리/ 한 명만 버스트면 버스트 패배
3. 동점 체크 - 딜러 및 플레이어 모두 버스트 / 블랙잭이 아니고 점수가 같으면 무승부
4. 점수 비교 - 딜러 및 플레이어 모두 버스트 / 블랙잭이 아니고 점수가 다르면 높은 쪽 승리

### 구현 기능

**1. [입력] 게임에 참여할 플레이어의 이름을 입력 받는다.**

- [조건] 쉼표 기준으로 이름을 구분한다.

- [예외] 쉼표 외의 다른 문자를 구분자로 한 경우 IllegalArgumentException 예외를 발생시킨다.

- [예외] 입력값이 공백일 경우 IllegalArgumentException 예외를 발생시킨다.

**2. [입력] 플레이별로 달아가며 베팅 금액을 입력받는다.**

- [예외] 입력값이 숫자가 아닐 경우 IllegalArgumentException 예외를 발생시킨다.

- [예외] 입력값이 0원이거나 음수일 경우 IllegalArgumentException 예외를 발생시킨다.

- [예외] 입력값이 공백일 경우 IllegalArgumentException 예외를 발생시킨다.

**3. [처리] 딜러와 플레이어에게 카드를 2장씩 나누어준다.**

- [조건] 덱은 1개만 사용한다.(A ~ K : 13 * 4 = 52장)

**4. [처리] 딜러와 플레이어의 최초 카드 합계를 구하며, 블랙잭 여부를 판단한다.**

- [조건] ACE가 11이어도 합이 21이 넘지 않는 경우에는 11, 넘는 경우에는 1로 계산한다.

**5. [출력] 카드 현황을 출력한다.**

- [출력] 딜러는 첫 번째 카드만 출력한다.

**6. [입력] 플레이어별로 카드 추가 지급 여부를 입력 받는다.**

- [처리] 사용자가 y를 입력하고, 사용자의 현재 카드 합계가 21미만일 경우에 카드를 지급한다.

- [조건] 모든 사용자가 추가 지급을 완료할 때까지 반복한다.

- [예외] y, n 외의 다른 문자일 경우 IllegalArgumentException 예외를 발생시킨다.

**7. [출력] 플레이어별로 지급 현황을 출력한다.**

**8. [출력] 딜러의 카드 합계가계가 17 이상이 될 때까지 추가 지급 현황을 출력한다.**

**9. [처리] 딜러와 플레이어의 최종 카드 합계를 계산한다.**

- [조건] ACE가 11이어도 합이 21이 넘지 않는 경우에는 11, 넘는 경우에는 1로 계산한다.

**10. [출력] 플레이어별 카드 현황 및 합계를 출력한다.**

**11. [처리] 최종 수익을 계산한다.**

- [수익 계산 - 플레이어]

    * 블랙잭: 베팅 금액의 1.5배 <br>
    * 승리: 베팅 금액의 1배<br>
    * 푸시(무승부): 베팅 금액 환급 (수익 없음)<br>
    * 패배: 베팅 금액 몰수 (베팅 금액만큼 마이너스)

- [수익 계산 - 딜러]

    * 플레이어들이 얻은 총 수익만큼 딜러는 손실을 입고, 잃은 만큼 수익을 얻는다.

**11. [출력] 최종 수익을 출력한다.**
