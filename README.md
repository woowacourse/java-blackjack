## 1️⃣ 개요

### 기능 흐름

```
1. 참가 인원의 이름을 입력 받는다.
↓
2. 참가자들에게 베팅 금액을 입력 받는다.
↓
3. 딜러와 참가자들에게 카드를 2장씩 나누어 주고, 각자의 패를 출력한다.
↓
┌─→ 4. 참가자들에게 카드를 한 장 더 받을지 입력 받는다. (y/n)
│            ↙ [y]                       ↘ [n]
│  5. 참가자에게 카드를 한 장 더    6. 참가자의 패를 출력한다.
│     나누어 주고, 패를 출력한다.           ↓ (다음 참가자)
└──────────────────────────┘      ←─────────┘
↓ (모든 참가자 완료)
7. 딜러의 패의 점수 합을 확인한다.
        ↙ [16 이하]             ↘ [17 이상]
딜러에게 카드를 한 장 더 나누어 준다.    (패스)
        └──────────────────────────┘
                     ↓
8. 각자의 최종 패와 점수 합을 출력한다.
↓
9. 승패 심판 및 최종 수익을 계산한다.
↓
10. 최종 수익 결과를 출력한다.
```

## 2️⃣ 구현 기능 목록
- [x] **1. [입력] 참가 인원의 이름을 입력 받는다.**
  - [x] [예외 처리] 이름에 숫자가 들어가는 경우, `IllegalArgumentException` 을 발생시키고, 재입력을 받는다.
  - [x] [예외 처리] 입력받은 이름 문자열에 쉼표 외의 특수문자가 포함된 경우, `IllegalArgumentException` 을 발생시키고, 재입력을 받는다.
  - [x] [예외 처리] 입력이 비어있는 경우, `IllegalArgumentException` 을 발생시키고, 재입력을 받는다.
  - [x] [예외 처리] 참가 인원이 16명 초과인 경우, `IllegalArgumentException` 을 발생시키고, 재입력을 받는다.


- [x] **2. [입력] 참가자들에게 배팅 금액을 입력 받는다.**
  - [x] [예외 처리] 입력이 비어있는 경우, `IllegalArgumentException` 을 발생시키고, 재입력을 받는다.
  - [x] [예외 처리] 숫자 외의 문자가 들어가는 경우, `IllegalArgumentException` 을 발생시키고, 재입력을 받는다.


- [x] **3. [중간 과정] 딜러와 참가자들에게 카드를 두 장씩 나누어 준다.**
  - [ ] [규칙] 두 장의 카드 숫자 합이 21일 경우, `블랙잭`이다.

  
- [x] **4. [출력] 각자의 패를 출력한다.**


- [x] **5. [입력] 참가자들에게 카드를 한 장 더 받을지 입력 받는다.**
  - [x] [예외 처리] y/n 외의 문자를 입력한 경우, `IllegalArgumentException` 을 발생시키고, 재입력을 받는다.
  - [x] [예외 처리] 입력이 비어있는 경우, `IllegalArgumentException` 을 발생시키고, 재입력을 받는다.

  
- [x] **6. [중간 과정] y인 경우, 참가자에게 카드를 한 장 더 나누어 준다.**


- [x] **7. [중간 과정] 참가자들이 가진 패의 점수 합을 계산한다.**
  - [x] [규칙] A를 포함한 패의 합이 21점이 넘을 경우, A를 1점으로 계산한다.
  - [x] [규칙] A를 포함한 패의 합이 21점이 넘지 않을 경우, A를 11점으로 계산한다.


- [x] **8. [출력] 4,5번 과정 이후 패를 출력한다.**


- [x] **9. [중간 과정] 딜러의 패의 점수 합을 계산한다.**


- [x] **10. [중간 과정] 딜러의 패의 점수 합이 16이하인 경우, 카드를 한 장 더 나누어 준다.**


- [x] **11. [중간 과정] 딜러의 패의 점수 합을 계산한다.**


- [x] **12. [출력] 각자의 패와 점수 합을 출력한다.**


- [x] **13. [중간 과정] 승패를 심판한다.**
  - [x] [규칙] 패의 점수 합이 21점을 초과하지 않으면서 상대보다 21점에 더 가까울 경우, 승으로 간주한다.
  - [x] [규칙] 패의 점수 합이 21점이 넘을 경우, 무조건 패로 간주한다.
  - [x] [규칙] 패의 점수 합이 21점<= && 동일한 경우, 공동 우승으로 간주한다.


- [ ] **14. [중간 과정] 최종 수익을 계산한다.**
  - [ ] [규칙] 참가자가 우승하는 경우, 참가자는 배팅 금액 만큼 수익을 얻고, 딜러는 같은 금액을 잃는다.
  - [ ] [규칙] 딜러가 우승하는 경우, 딜러는 참가자의 배팅 금액 만큼 수익을 얻고, 참가자는 같은 금액을 잃는다.
  - [ ] [규칙] 참가자의 처음 두 장의 카드 합이 21일 경우, 블랙잭이 되어 배팅 금액의 1.5 배를 딜러에게 받는다.
  - [ ] [규칙] 딜러와 참가자가 모두 동시에 블랙잭인 경우, 참가자는 배팅한 금액을 돌려받는다.
  - [ ] [규칙] 딜러가 21을 초과하는 경우, 그 시점까지 남아 있던 참가자들은 가지고 있는 패에 상관 없이 승리해 배팅 금액을 받는다.


- [ ] **15. [출력] 최종 수익을 출력한다.**


## 3️⃣ 입출력 요구 사항

### 실행 결과 예시

```
게임에 참여할 사람의 이름을 입력하세요.(쉼표 기준으로 분리)
pobi,jason

pobi의 배팅 금액은?
10000

jason의 배팅 금액은?
20000

딜러와 pobi, jason에게 2장을 나누었습니다.
딜러: 3다이아몬드
pobi카드: 2하트, 8스페이드
jason카드: 7클로버, K스페이드

pobi는 한장의 카드를 더 받겠습니까?(예는 y, 아니오는 n)
y
pobi카드: 2하트, 8스페이드, A클로버
pobi는 한장의 카드를 더 받겠습니까?(예는 y, 아니오는 n)
n
pobi카드: 2하트, 8스페이드, A클로버
jason은 한장의 카드를 더 받겠습니까?(예는 y, 아니오는 n)
n
jason카드: 7클로버, K스페이드

딜러는 16이하라 한장의 카드를 더 받았습니다.

딜러 카드: 3다이아몬드, 9클로버, 8다이아몬드 - 결과: 20
pobi카드: 2하트, 8스페이드, A클로버 - 결과: 21
jason카드: 7클로버, K스페이드 - 결과: 17

## 최종 수익
딜러: 10000
pobi: 10000 
jason: -20000
```
