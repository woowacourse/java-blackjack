### 카드 기능
- Ace는 기본 점수를 1로 계산한다.
    - Ace를 11로 처리했을 때 버스트 되지 않는 경우 11로 취급한다.
    - Ace를 11로 처리했을 때 버스트되는 경우 1로 취급한다.
- 최종 점수에서 11로 승격시킬 지 판단한다.
- King, Queen, Jack은 10으로 계산한다.
- 카드 문양은 다이아몬드, 스페이드, 클로버, 하트로 총 4가지가 존재한다.

### 입력 기능
- 게임에 참여할 사람의 이름을 쉼표 기준으로 입력받을 수 있다.
- 사용자 이름은 5글자 이하로 제한한다.
- 잘못된 이름을 입력하면 다시 입력받는다.
- 사용자는 카드를 더 받을 건지 결정할 수 있다.
  - 예는 y, 아니오는 소문자로만 구분한다.
  - y, n이 아닌 값을 입력하면 다시 입력받는다.
- 각 사용자에게 배팅 금액을 입력받는다.
  - 음수인 경우 예외를 발생시킨다.
  - 숫자가 아닌 경우 예외를 발생시킨다.

### Deal 기능
- 첫 라운드에서 딜러를 포함하여, 모든 참가자가 카드 2장을 가진다. (딜러가 참가자에게 나눠주고, 딜러가 자신의 카드를 가져감)

### Hit 기능
- 첫 라운드 이후 플레이어에게 카드를 하나 더 뽑을 건지 물어본다.
- 이때 플레이어의 카드 합계가 21을 초과한다면 물어보지 않는다.
- y를 입력하면 플레이어가 카드를 한 장 더 받는다.
- n를 입력하면 다음 플레이어에게 넘어간다.

### 게임 종료 조건
- 다음 조건을 만족하면 게임을 종료하고, 통계를 낸다.
  - 모든 플레이어가 버스트이거나 스탠드를 선언한 경우
  - 딜러의 카드 합계가 17 이상일 경우

### 승패 판정
- 딜러보다 카드 최종 점수가 높은 플레이어는 승리한다.
- 딜러보다 카드 최종 점수가 같은 플레이어는 무승부이다.
- 딜러보다 카드 최종 점수가 낮은 플레이어는 패배한다.
- 플레이어가 버스트라면, 무조건 패배한다.
- 플레이어가 버스트이고, 딜러가 버스트이면 딜러가 승리한다.


### 배팅 기능
- 블랙잭 여부를 판단한다.
- 배팅 금액에 따라 수익을 계산한다.
  - 딜러와 플레이어 모두 블랙잭인 경우 → 0배 수익
  - 플레이어만 블랙잭인 경우 → 1.5배 수익
  - 딜러만 블랙잭인 경우 → 딜러 승리
  - 승리인 경우 → 1배 수익
  - 무승부인 경우 → 0배 수익
  - 딜러가 21을 초과하면 남은 플레이어는 승리처리한다.

## 도메인 용어 정의

Deal - 게임 시작 직수, 처음 카드를 두장 나눠주는 행위.
Hit - 카드 합이 21이하여서, 카드를 한 장 더 뽑는 행위
CardDenomination - 카드 숫자
CardEmblem - 카드 문양(엠블렘)
Card - 카드 한 장
Player - 게임 플레이어
Dealer - 게임 딜러
CardDeck - 딜러가 가지는 카드 덱
CardBundle - 게임 참가자가 가지는 카드 묶음.
ResultAnalyzer - 게임 통계 분석기
HandResult - 손패 결과
GameResult - 게임 결과 (승, 패, 무)
