# 블랙잭

## 구현할 기능 목록

### 카드
- [x] 카드는 무늬(Suit)와 숫자(Rank)를 가진다
- [x] 무늬는 스페이드, 하트, 다이아몬드, 클로버 4가지이다
- [x] 숫자는 A, 2~10, J, Q, K이며 A는 1 또는 11로 계산한다

### 덱
- [x] 52장의 카드로 구성된다
- [x] 게임 시작 시 카드를 섞는다
- [x] 카드를 한 장씩 뽑을 수 있다

### 참여자
- [x] 딜러와 플레이어는 카드를 받을 수 있다
- [x] 딜러와 플레이어는 보유한 카드의 점수를 계산할 수 있다
- [x] 플레이어는 21 미만이면 카드를 더 받을 수 있다
- [x] 딜러는 16 이하이면 카드를 더 받아야 한다
- [x] ACE를 포함할 때 버스트가 나지 않으면 11로, 아니면 1로 계산한다

### 게임 진행
- [x] 게임 시작 시 딜러와 플레이어에게 각 2장씩 나눠준다
- [x] 초기 딜 시 딜러는 카드 1장만 공개한다
- [x] 플레이어는 순서대로 히트(y) 또는 스테이(n)를 선택한다
- [x] 플레이어가 히트 시 카드를 받고 현재 카드를 출력한다
- [x] 플레이어가 처음부터 스테이 시 초기 카드를 출력한다
- [x] 플레이어가 히트 후 스테이 시 카드를 출력하지 않는다
- [x] 모든 플레이어 턴 종료 후 딜러 턴을 진행한다

### 결과 계산
- [x] 딜러와 플레이어의 최종 점수를 비교해 승패를 결정한다
- [x] 버스트(21 초과) 시 해당 참여자가 패배한다
- [x] 딜러 결과는 플레이어 결과의 역으로 계산한다

### 입출력
- [x] 참여자 이름을 쉼표 기준으로 입력받는다
- [x] 히트 여부를 y/n으로 입력받는다
- [x] 초기 딜 결과를 출력한다
- [x] 최종 카드 및 점수를 출력한다
- [x] 최종 승패를 출력한다

## 프로그래밍 요구 사항

### 기본 요구 사항
- [x] 자바 코드 컨벤션을 지키며 프로그래밍한다 (Java Style Guide 기준)
- [x] indent depth를 1까지만 허용한다
- [x] 3항 연산자를 사용하지 않는다
- [x] else 예약어를 사용하지 않는다 (switch/case 포함)
- [x] 모든 기능을 TDD로 구현하며 단위 테스트가 존재한다 (UI 로직 제외)
- [x] 함수(메서드)의 길이가 10라인을 넘지 않는다
- [x] 배열 대신 컬렉션을 사용한다
- [x] 모든 원시 값과 문자열을 포장한다
- [x] 줄여 쓰지 않는다 (축약 금지)
- [x] 일급 컬렉션을 사용한다

### 추가 요구 사항
- [x] 모든 엔티티를 작게 유지한다
- [x] 3개 이상의 인스턴스 변수를 가진 클래스를 사용하지 않는다
- [x] 딜러와 플레이어에서 발생하는 중복 코드를 제거한다

## 팀 규칙

### 테스트 단위 기준 규칙 (나무)
- [x] 구현할 기능 목록 하나가 완성되면 해당 단위의 테스트를 작성한다
- [x] 기능 수행 성공 케이스 테스트를 먼저 작성한다
- [x] 기능 수행 실패(예외) 케이스 테스트를 작성한다
- [x] 경계값에 대한 테스트를 작성한다
- [x] 하나의 테스트 안에서 여러 기능을 테스트하지 않는다

### 리팩터링 우선순위 규칙 (캐리)
- [x] 메서드의 매개변수가 3개 이상이면 관련 변수를 하나의 객체로 묶어 전달한다
- [x] 연관성을 먼저 식별 후 데이터를 구조화하고 시그니처를 교체한다
- [x] 연관 없는 데이터를 개수 줄이기 위해 억지로 하나의 객체에 넣지 않는다

### 테스트 어려움 대응 규칙 (티뉴)
- [x] 메서드가 2개 이상의 도메인 로직을 포함하면 각 로직을 별도 메서드로 분리한다
- [x] 분리한 메서드에 외부 의존이 있으면 파라미터로 꺼낸다
- [x] 원시 타입을 객체로 바꿔 각 객체가 자신의 로직을 갖게 한다
- [x] 개별 메서드가 검증되었다면 상위 메서드는 테스트 대상에서 제외한다
- [x] 판단 로직이 없는 private 메서드를 테스트하기 위해 접근 제어자를 수정하지 않는다

### 테스트 제외 기준 규칙 (아이큐)
- [x] 비즈니스 판단 로직이 없는 코드는 단위 테스트 대상에서 제외한다
- [x] 자바 표준 API(List.add(), Enum.values() 등)의 신뢰성을 검증하지 않는다
