# java-blackjack

우아한테크코스 1주차 블랙잭 미션 저장소

콘솔에서 실행하는 블랙잭 게임입니다.

## 블랙잭 규칙

- **목표**: 딜러보다 21에 가깝게, 21을 초과하지 않도록 카드 합계를 맞추는 것
- **카드 점수**: 2~10은 숫자 그대로, J/Q/K는 10점, A는 1 또는 11 (버스트를 피하는 방향으로 자동 계산)
- **블랙잭**: 처음 2장으로 21을 만든 경우 (A + 10/J/Q/K)
- **버스트**: 21을 초과한 경우 즉시 패배
- **딜러**: 16 이하는 반드시 히트, 17 이상이면 스탠드


## 주요 클래스 설명

| 클래스 | 역할 |
|--------|------|
| **GameController** | 전체 게임 흐름: 참가자 설정 → 초기 배분 → 플레이어 턴 → 딜러 턴 → 결과 출력 |
| **GameService** | 초기 배분, 카드 딜링, 승패 판정 |
| **Player** | 카드 수령, 점수 계산, 버스트/블랙잭/히트 여부 판단, 에이스 보너스 계산 |
| **CardDeck** | ShuffleStrategy로 덱 셔플 후, 카드 딜 |
| **AppConfig** | Controller, Service, View 등을 생성하고 조립 |

## 게임 플로우

1. **참가자 입력**: 쉼표로 구분된 이름 입력 (예: `pobi,jason`)
2. **초기 배분**: 참가자와 딜러에게 각각 2장씩 배분
3. **플레이어 턴**: 각 참가자는 `y`(히트) / `n`(스탠드) 입력으로 카드 추가 여부 결정
4. **딜러 턴**: 합계 16 이하면 자동 히트, 17 이상이면 스탠드
5. **결과**: 최종 카드와 점수 출력 후 승/패/무 및 딜러 전적 출력

## 트러블슈팅

1. **Ace 판별 시점**: 에이스를 1로 둘지 11로 둘지 언제 결정할지
2. **테스트 코드 범위**: 비즈니스 로직 위주로 테스트, 과도한 세부 검증 지양
3. **카드 셔플 검증**: 셔플 결과가 겹치는 경우 검증 방식
4. **카드 리스트 싱글톤**: 덱 공유 여부 및 생명주기
5. **테스트 전용 메서드**: CardDeck 등에서 기능상 필요 없지만 테스트를 위해 덱 상태 조회 메서드를 두는 것에 대한 고민
