## 🃏 블랙잭 요구사항

블랙잭 게임은 딜러와 플레이어 중 카드의 합이 21 또는 21에 가장 가까운 숫자를 가지는 쪽이 이기는 게임이다.

- 카드의 숫자 계산은 카드 숫자를 기본으로 한다.
    - 예외로 Ace는 1 또는 11로 계산한다.
    - King, Queen, Jack은 각각 10으로 계산한다.

- 플레이어는 게임을 시작할 때 배팅 금액을 정해야 한다.
- 게임을 시작하면 플레이어는 두 장의 카드를 지급 받으며, 두 장의 카드 숫자를 합쳐 21을 초과하지 않으면서 21에 가깝게 만들면 이긴다.
    - 21을 넘지 않을 경우 원한다면 얼마든지 카드를 계속 뽑을 수 있다.
    - 단, 카드를 추가로 뽑아 21을 초과할 경우 배팅 금액을 모두 잃게 된다.
- 처음 두 장의 카드 합이 21일 경우 블랙잭이 되면 베팅 금액의 1.5 배를 딜러에게 받는다.
    - 딜러와 플레이어가 모두 동시에 블랙잭인 경우 플레이어는 베팅한 금액을 돌려받는다.
- 딜러는 처음에 받은 2장의 합계가 16 이하이면 반드시 1장의 카드를 추가로 받아야 하고, 17점 이상이면 추가로 받을 수 없다.
    - 딜러가 21을 초과하면 그 시점까지 남아 있던 플레이어들은 가지고 있는 패에 상관 없이 승리해 베팅 금액을 받는다.
- 게임을 완료한 후 모든 참가자의 승패와 수익을 출력한다.

## ✅ 구현 기능 목록

### 입력

- [x] 참여할 플레이어 이름 요청
    - 중복된 이름이 입력된 경우 예외 발생
    - 딜러 제외 참여 인원이 1명-7명이 아닌 경우 예외 발생
        - 최대 인원 결정
          이유: [표준 카지노 룰](https://namu.wiki/w/%EB%B8%94%EB%9E%99%EC%9E%AD(%ED%94%8C%EB%A0%88%EC%9E%89%20%EC%B9%B4%EB%93%9C)#:~:text=%EC%B9%B4%EB%93%9C%EC%9D%98%20%ED%95%A9%EC%9D%B4%2021%EC%A0%90%20%EB%98%90%EB%8A%94%2021%EC%A0%90%EC%97%90%20%EA%B0%80%EC%9E%A5%20%EA%B0%80%EA%B9%9D%EA%B2%8C%20%EB%A7%8C%EB%93%A4%EB%A9%B4%20%EC%8A%B9%EB%A6%AC%ED%95%98%EB%A9%B0%2C%2021%EC%9D%84%20%EC%B4%88%EA%B3%BC%ED%95%98%EB%A9%B4%20%ED%8C%A8%EB%B0%B0%ED%95%9C%EB%8B%A4.%20%EC%82%AC%EC%9A%A9%ED%95%98%EB%8A%94%20%EC%B9%B4%EB%93%9C%EB%8A%94%20%EC%82%AC%EC%A0%81%EC%9D%B8%20%EA%B2%8C%EC%9E%84%EC%97%90%EC%84%9C%EB%8A%94%20%EC%A1%B0%EC%BB%A4%EB%A5%BC%20%EC%A0%9C%EC%99%B8%ED%95%9C%2052%EC%9E%A5%EC%9D%84%2C%20%EC%B9%B4%EC%A7%80%EB%85%B8%EC%97%90%EC%84%9C%EB%8A%94%20%EC%97%AC%EB%9F%AC%20%EB%B2%8C(%EB%B3%B4%ED%86%B5%206%EB%B2%8C)%EC%9D%84%20%EC%82%AC%EC%9A%A9%ED%95%98%EB%A9%B0%2C%20%EC%B0%B8%EA%B0%80%ED%95%98%EB%8A%94%20%EC%9D%B8%EC%9B%90%EC%9D%80%202~8%EB%AA%85%EC%9D%B4%EB%8B%A4.)
    - 이름 길이가 1-5자 사이가 아닐 경우 예외 발생

- [x] 배팅 금액 요청
    - 숫자가 아닐 경우 예외 발생
    - 금액이 1000원 단위로 나누어 떨어지지 않을 경우 예외 발생
    - 금액이 1000-300000원 사이가 아닌 경우 예외 발생
        - 배팅 금액 한도 결정
          이유: [강원랜드 배팅액](http://www.wikileaks-kr.org/news/articleView.html?idxno=148517#:~:text=%EA%B0%95%EC%9B%90%EB%9E%9C%EB%93%9C%20%EC%B9%B4%EC%A7%80%EB%85%B8%20%EC%9D%BC%EB%B0%98%20%EC%98%81%EC%97%85%EC%9E%A5%20%ED%85%8C%EC%9D%B4%EB%B8%94%20%EA%B2%8C%EC%9E%84%EC%9D%80%20%EC%B5%9C%EA%B3%A0%20%EB%B2%A0%ED%8C%85%20%ED%95%9C%EB%8F%84%2030%EB%A7%8C%20%EC%9B%90%2C%20%EC%B5%9C%EC%A0%80%20%EB%B2%A0%ED%8C%85%EA%B8%88%EC%95%A1%EC%9D%80%201000%EC%9B%90%EC%9C%BC%EB%A1%9C%20%EA%B7%9C%EC%A0%9C%ED%95%98%EA%B3%A0%20%EC%9E%88%EB%8B%A4.)

- [x] 카드 더 받을지 여부(`예: y`, `아니오: n`) 요청
    - 입력값이 `y(Y)` 또는 `n(N)`이 아닌 경우 예외 발생

### 출력

- [x] 플레이어가 소지한 카드 패 출력
- [x] 카드 점수 합계 출력
- [x] 딜러의 카드 드로우 여부 출력
- [x] 최종 수익 출력

### 카드 생성

- [x] 총 52장의 카드 생성 후 셔플
    - 문양 : 하트, 스페이드, 클로버, 다이아몬드 4가지
    - 숫자 : 각 문양 당 2-10, J, Q, K, A 총 13가지

### 카드 분배

- [x] 카드 2장 분배 경우
    - 시작 시 모든 플레이어에게 분배
- [x] 카드 1장 분배 경우
    - 딜러를 제외한 플레이어의 카드 총합이 21 미만 + 플레이어의 히트 요청 시 분배
    - 딜러의 카드 총합이 17 미만일 때 분배

### 카드 점수 계산

- [x] 플레이어가 소지한 모든 카드의 점수 총합
    - 소지 패에 A가 포함 **된** 경우
        - A를 1 또는 11로 계산
            - A를 1로 계산하는 경우: 하나의 A를 11로 계산하면 버스트가 될 때
        - 21을 초과하지 않는 가장 높은 점수로 계산

### 버스트 판단

- [x] 플레이어의 카드 총합이 21을 초과한 경우 버스트(패배)

### 승패 판단

- [x] 플레이어 패배 상황
    - [x] 플레이어가 버스트된 경우
    - [x] 플레이어의 카드 총합이 딜러보다 낮을 경우
- [x] 무승부 상황
    - [x] 딜러와 플레이어의 카드 총합이 같을 경우
- [x] 플레이어 승리 상황
    - [x] 플레이어의 카드 총합이 딜러보다 높을 경우
    - [x] 플레이어는 버스트가 아니지만, 딜러가 버스트된 경우
- [x] 딜러의 승/패/무 횟수 계산

### 플레이어 수익 계산

- [x] 플레이어가 진 경우
    - [x] 수익: 배팅 금액 반환 X, 수익은 -배팅금
- [x] 플레이어가 이긴 경우
    - [x] 블랙잭으로 이긴 경우(처음 2장의 합이 21)
        - [x] 수익: 배팅 금액 x 1.5배 수익
    - [x] 그냥 이긴 경우
        - [x] 수익: 배팅 금액과 동일한 수익
- [x] 플레이어와 딜러가 무승부인 경우
    - 플레이어와 딜러 모두 블랙잭인 경우
    - 플레이어 점수와 딜러 점수가 동일한 경우
    - [x] 수익: 배팅 금액 반환 O, 수익은 0

### 딜러 수익 계산

- [x] 수익: (패배한 플레이어 수익의 합) - (승리한 플레이어 수익의 합)
