# 기능 구현 목록

### 입력

- 게임에 참여할 플레이어 이름을 입력받는다. (쉼표(,)로 구분)
- 플레이어별 배팅 금액을 입력받는다.
- 각 플레이어에 대해서 카드를 더 받을지 입력받는다. (예: y, n)

### 카드

- 카드의 모양은 하트, 다이아몬드, 클로버, 스페이드로 구성된다.
- 카드의 랭크는 2부터 10 그리고 Ace, Jack, Queen, King가 존재한다.

### 덱

- 52장의 카드로 구성된다.
- 덱에서 카드를 뽑는 기능 (맨 뒤에 있는 카드를 return하고 덱에서 제거한다.)
- 덱에서 카드를 섞는 기능

### 패

- 패는 카드의 리스트로 구성된다.
- 패의 점수 합을 갖는다.
- 패의 점수를 계산하는 기능
    - Ace는 1 또는 11로 자동 최적으로 계산
    - 2부터 10까지의 카드는 해당 숫자만큼 계산된다.
    - Jack, Queen, King은 각각 10으로 계산된다.
- 패에 카드를 추가하는 기능

### 돈

- 0원 이상인지 검증한다.

### 참가자

- 패를 가지고 있다.
- 첫 손 패는 2장으로 시작된다.
- 드로우 기능

#### 딜러

- 단 딜러의 첫 패는 한 장은 공개되고, 한 장은 비공개로 시작된다.

#### 플레이어

- 배팅을 한다.
    - 시작 패의 합이 21인 경우 배팅금액의 1.5배를 얻는다.
    - 승리 시 배팅금액의 2배를 얻는다.
    - 패배 시 배팅금액을 잃는다.

### 배팅 정보

- 플레이어와 배팅금액을 가진다.

### 게임 결과

- 최종 승패를 계산한다.
- 수익을 계산한다.
- 플레이어, 승패, 수익을 저장한다.

### 점수 결과

- 이름을 갖는다.
- 패를 갖는다.

### 최종 승패 결과

- 딜러 승패 카운트
- 플레이어 승패 정보

### 핵심 로직 (service)

- 게임 초기화 로직 (덱 생성, 섞기, 플레이어 및 딜러에게 카드 분배)
- 각 플레이어의 히트, 스탠드 로직
    - 플레이어가 카드를 더 받을지 묻는다.
- 딜러의 히트, 스탠드 로직
    - 딜러의 점수가 17 이상이 될 때까지 카드를 더 받는다.
- 승패에 따라 수익을 계산한다.

### 예외 상황

- 닉네임이 빈 문자열인 경우
- 플레이어의 입력이 y혹은 n가 아닌 값인 경우
- 인원이 26명 이상인 경우(덱의 카드가 부족함)
- 배팅 금액이 숫자가 아닌 경우

### 출력

- 첫 패를 나눠주는 멘트를 작성한다.
- 딜러와 플레이어 각각의 손 패를 출력한다.
- 각 플레이어가 카드를 더 받을지 묻는 멘트를 출력한다.
- 플레이어가 카드를 더 받는 경우, 받은 카드와 현재 손 패를 출력한다.
- 플레이어가 카드를 한장도 더 받지 않고 종료한 경우, 손 패를 출력한다.
- 딜러가 카드를 더 받는 경우, 더 받았다는 멘트를 출력한다.
- 모든 플레이어 및 딜러의 손패와 결과를 출력한다.
- 최종 수익을 출력한다.

### 실행 예시

```
게임에 참여할 사람의 이름을 입력하세요.(쉼표 기준으로 분리)
pobi,jason

pobi의 배팅 금액은?
10000

jason의 배팅 금액은?
20000

딜러와 pobi, jason에게 2장을 나누었습니다.
딜러카드: 3다이아몬드
pobi카드: 2하트, 8스페이드
jason카드: 7클로버, K스페이드

pobi는 한장의 카드를 더 받겠습니까?(예는 y, 아니오는 n)
y
pobi카드: 2하트, 8스페이드, A클로버
pobi는 한장의 카드를 더 받겠습니까?(예는 y, 아니오는 n)
n
pobi카드: 2하트, 8스페이드, A클로버
jason는 한장의 카드를 더 받겠습니까?(예는 y, 아니오는 n)
n
jason카드: 7클로버, K스페이드

딜러는 16이하라 한장의 카드를 더 받았습니다.

딜러카드: 3다이아몬드, 9클로버, 8다이아몬드 - 결과: 20
pobi카드: 2하트, 8스페이드, A클로버 - 결과: 21
jason카드: 7클로버, K스페이드 - 결과: 17

## 최종 수익
딜러: 10000
pobi: 10000 
jason: -20000
```
