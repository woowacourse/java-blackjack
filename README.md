# java-blackjack

## 목표
- 마감시간 지키기(매우 중요!!!!!)
- 테스트 사용을 제외한 모든 getter 제거하기(구현 후 리팩토링 하면서 시도하기)
- 경계 테스트, 예외 테스트 작성하기
- 토론 활동으로 도출된 규칙 적용하기!
  ```
  1. 책임 분리 기준
    - 상태를 가진 객체 이외에 세부 연산 로직이 존재하면 책임을 분리한다.
    - 역할에 따른 기능을 읽었을 때 어떤 객체에게 메시지를 던질지 고민한다. 
  2. 변경 범위 제한 규칙
    - 기능 하나 추가로 메서드가 5개 이상 변경되면 무조건 재설계
    - 메서드 4개 이하의 수정이면 재설계 고민
  3. 테스트와 설계의 관계 규칙
    - 세부 기능 구현 사항 변경으로 인해 테스트를 수정해야 한다면 테스트를 수정한다.
    - 외부에서 기대하는 동작이 변경되지 않았는데 테스트를 수정해야 한다면 코드를 변경한다.
  ```
  
### 추가 기능 요구 사항
- [x] 카드가 2장이고, 점수 합계가 21점이면 `블랙잭`이다.
- [x] 플레이어의 배팅 금액을 입력 받는다.
  ```
  usher의 배팅 금액은?
  10000
  ```
- [x] 배팅 금액은 최소 1,000원이며 최대 50,000원이다. 
  - [x] 이외의 금액이 들어오면 예외를 발생시킨다.
- [x] 플레이어 `승리`이면, 배팅 금액의 1배이다.
  - [x] `블랙잭`으로 승리하면 배팅 금액의 1.5배이다.
- [x] `무승부`이면 배팅 금액을 돌려 받는다.
  - 플레이어와 딜러 모두 블랙잭이면 무승부이다.
- [x] 플레이어 `패배`이면 배팅 금액을 잃는다. 
- [x] 최종 수익을 출력한다.
  ```
  ## 최종 수익
  딜러: 10000
  pobi: 10000
  jason: -20000
  ```

# 사이클 1
## 기능 요구 사항
### 기능
- [x] 플레이어는 2명 이상, 5명 이하여야 한다.
- [x] 카드 덱은 총 52장으로 이루어져 있다.
  - 카드 종류는 스페이드, 하트, 클로버, 다이아몬드가 있다.
  - 숫자는 2부터 10까지 있고, 문자는 A,J,Q,K가 있다.
- [x] 카드 덱의 순서를 무작위로 섞는다.
- [x] Ace는 1 또는 11로 계산할 수 있다.
- [x] King, Queen, Jack은 각각 10으로 계산한다.
- [x] 플레이어와 딜러는 무작위 카드 2장을 받는다.
- [x] 플레이어는 21이 넘지 않으면 얼마든지 카드를 계속 뽑을 수 있다.
- [x] 딜러는 2장의 합계가 16점 이하면 반드시 1장을 카드를 추가로 받는다.
- [x] 딜러는 2장의 합계가 17점 이상이면 추가로 받을 수 없다.
- [x] 카드의 숫자 합을 계산한다.
- [x] 합이 21이 넘을 경우 버스트로 판정한다.
  - [x] 딜러가 버스트일 경우 플레이어가 승리한다.
  - [x] 플레이어가 모두 버스트일 경우 딜러가 승리한다.
  - [x] 플레이어 일부가 버스트가 아닐 경우, 버스트인 플레이어만 패배로 판정한다.

### 입력
- [x] 참여할 플레이어의 이름을 입력한다.
  - [x] 쉼표 기준으로 분리한다.
  - [x] 이름 길이는 최소 2자 이상, 5자 이하로 입력한다.
  ```
    게임에 참여할 사람의 이름을 입력하세요.(쉼표 기준으로 분리)
    pobi,jason
  ```
- [x] 각 플레이어에게 카드를 더 받을지 입력 받는다.
    ```
    pobi는 한장의 카드를 더 받겠습니까?(예는 y, 아니오는 n)
    y
    ```

### 출력
- [x] 첫 시작 카드 현황을 출력한다.
  - 딜러의 카드는 한장만 출력하고 플레이어의 카드는 모두 출력한다.
    ```
    딜러와 pobi, jason에게 2장을 나누었습니다.
    딜러카드: 3다이아몬드
    pobi카드: 2하트, 8스페이드
    jason카드: 7클로버, K스페이드
    ```
- [x] 플레이어의 카드 현황을 출력한다.
  ```
  pobi카드: 2하트, 8스페이드, A클로버
  ```
- [x] 딜러의 카드 추가 발급 여부를 출력한다.
  - 딜러의 카드가 16 이하라면 다음과 같은 메시지를 출력한다.  
    ```
    딜러는 16이하라 한장의 카드를 더 받았습니다.
    ```
- [x] 딜러와 플레이어의 카드 현황 및 점수와 최종 승패를 출력한다.
  ```
  딜러카드: 3다이아몬드, 9클로버, 8다이아몬드 - 결과: 20
  pobi카드: 2하트, 8스페이드, A클로버 - 결과: 21
  jason카드: 7클로버, K스페이드 - 결과: 17
    
  ## 최종 승패
  딜러: 1승 1패
  pobi: 승
  jason: 패
  ```

## **프로그래밍 요구 사항**

- 자바 코드 컨벤션을 지키면서 프로그래밍한다.
    - 기본적으로 [Java Style Guide](https://github.com/woowacourse/woowacourse-docs/tree/master/styleguide/java)을 원칙으로 한다.
- indent(인덴트, 들여쓰기) depth를 2를 넘지 않도록 구현한다. 1까지만 허용한다.
    - 예를 들어 while문 안에 if문이 있으면 들여쓰기는 2이다.
    - 힌트: indent(인덴트, 들여쓰기) depth를 줄이는 좋은 방법은 함수(또는 메서드)를 분리하면 된다.
- 3항 연산자를 쓰지 않는다.
- else 예약어를 쓰지 않는다.
    - else 예약어를 쓰지 말라고 하니 switch/case로 구현하는 경우가 있는데 switch/case도 허용하지 않는다.
    - 힌트: if문에서 값을 반환하는 방식으로 구현하면 else 예약어를 사용하지 않아도 된다.
- 모든 기능을 TDD로 구현해 단위 테스트가 존재해야 한다. 단, UI(System.out, System.in) 로직은 제외
    - 핵심 로직을 구현하는 코드와 UI를 담당하는 로직을 구분한다.
    - UI 로직을 InputView, ResultView와 같은 클래스를 추가해 분리한다.
- 함수(또는 메서드)의 길이가 10라인을 넘어가지 않도록 구현한다.
    - 함수(또는 메소드)가 한 가지 일만 하도록 최대한 작게 만들어라.
- 배열 대신 컬렉션을 사용한다.
- 모든 원시 값과 문자열을 포장한다.
- 줄여 쓰지 않는다(축약 금지).
- 일급 컬렉션을 쓴다.

### **추가된 요구 사항**

- 모든 엔티티를 작게 유지한다.
- 3개 이상의 인스턴스 변수를 가진 클래스를 쓰지 않는다.
- 딜러와 플레이어에서 발생하는 중복 코드를 제거해야 한다.

## 페어와 정한 우리들만의 규칙
```
(테스트 단위 기준) 메서드 단위로 테스트한다.

(If-Then) 테스트코드 작성이 어렵다. 
         → 테스트가 가능하게 기존 코드를 리팩토링한다.
          테스트마다 결과가 달라진다.
         → 분리해서 랜덤 값을 고정 값으로 설정한 뒤 테스트.

(우선순위) 리팩터링 순서
		1. 결합도(Coupling)를 낮춘다.
		2. 중복코드 제거(메서드 분리)
		3. else 예약어 안쓰기

(금지) 입출력기능(Scanner, sout)자체는 테스트하지 않는다.
```
