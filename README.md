# java-blackjack

블랙잭 미션 저장소

# ✅ 구현할 기능 목록

## 게임 준비

---

### 참여자 이름 입력받기

- [ ]  게임에 참여할 플레이어의 이름 입력받기
    - [ ]  안내 메시지: `게임에 참여할 사람의 입력을 입력하세요.`
    - [ ]  입력 형태: `pobi,jason`

### 입력값 기본 파싱 및 검증

- [ ]  입력의 앞뒤 공백 제거하기 (trim)
- [ ]  입력이 공백인 경우 재입력
    - [ ]  예외 메시지: `[ERROR] 입력값이 비어있거나 공백입니다.`

### 플레이어 이름 파싱

- [ ]  `,` 기준으로 이름을 나누기
    - [ ]  이름 사이 공백 허용하고 이를 제거함

### 플레이어 이름 객체 생성 및 검증

- [ ]  이름이 공백일 경우 재입력
    - [ ]  예외 메시지: `[ERROR] 이름은 공백이 될 수 없습니다.`
- [ ]  이름 길이가 10글자가 넘을 경우 재입력
    - [ ]  예외 메시지: `[ERROR] 이름 길이는 최대 10글자입니다.`

### 전체 플레이어 명단 검증

- [ ]  중복되는 이름이 있을 경우 재입력
    - [ ]  예외 메시지: `[ERROR] 중복되는 이름이 존재합니다.`

## 게임 초기 진행

---

### 덱 섞기

- [ ]  게임 시작과 동시에 카드 섞기
    - [ ]  52장의 카드

### 초기 카드 분배

- [ ]  게임 시작 시 모든 사람에게 2장씩 주기
    - [ ]  딜러와 모든 플레이어에게 전달
    - [ ]  딜러 > 플레이어 이름 입력 순으로 전달

### 초기 카드 정보 출력

- [ ]  초기 카드 배분 결과 출력하기
    - [ ]  출력 형태: `딜러와 {player1}, {player2}에게 2장을 나누었습니다.`
    - [ ]  딜러는 1장의 카드 정보만 출력
        - [ ]  출력 형태 예: `딜러카드: 3다이아몬드`
    - [ ]  플레이어는 2장의 카드 모두 출력
        - [ ]  출력 형태 예: `pobi카드: 2하트, 8스페이드, A클로버`

## 플레이어의 카드 추가 수령

---

### 카드 추가 수령 여부 입력

- [ ]  플레이어에게 카드 추가 수령 여부 입력받기
    - [ ]  안내 메시지: `{playerName}는 한장의 카드를 더 받겠습니까?(예는 y, 아니오는 n)`
    - [ ]  입력 형태: `y` 혹은 `n`
- [ ]  [입력값 기본 검증](https://www.notion.so/31938a2a7fbd80b8a8d0df5e79e05871?pvs=21) 거치기
- [ ]  y, n이 아닌 경우 재입력
    - [ ]  예외 메시지: `[ERROR] y 혹은 n으로 입력해야 합니다.`

### 카드 지급 및 누적 카드 정보 출력

- [ ]  `y`를 입력한 경우, 카드 1장을 추가로 지급하기
- [ ]  현재 해당 플레이어의 카드 목록 출력하기
    - [ ]  출력 형태 예: `pobi카드: 2하트, 8스페이드, A클로버`

### 버스트 판별

- [ ]  누적 카드 합계 계산하기
- [ ]  21점 초과일 경우, 버스트로 판단하기

### 카드 추가 수령 종료 조건 판별

- [ ]  다음 조건 중 하나일때까지 반복하기
    - [ ]  `n`를 입력할때
    - [ ]  버스트가 될때

## 딜러의 카드 추가 수령

---

### 딜러 카드 수령 조건 판별

- [ ]  누적 카드 합계 계산하기
- [ ]  누적 카드 합계가 16이하이면, 추가 카드 받기
    - [ ]  안내 메시지: `딜러는 16이하라 한장의 카드를 더 받았습니다.`
- [ ]  누적 카드 합계가 17 이상이면, 딜러의 추가 카드 수령 종료

## 게임 결과

---

### 최종 점수 계산

- [ ]  참여자의 카드 합계 계산
    - [ ]  Ace는 11로 디폴트, 21점 초과시 1로 계산

### 참여자의 최종 카드 및 결과 출력

- [ ]  각 참여자의 카드 목록과 최종 점수 출력
    - [ ]  출력 형태 예: `딜러카드: 3다이아몬드, 9클로버, 8다이아몬드 - 결과: 20`

### 플레이어들과 딜러 승패계산

- [ ]  플레이어의 버스트 확인
    - [ ]  딜러 무조건 승
- [ ]  딜러의 버스트 확인
    - [ ]  플레이어가 버스트가 아닐때, 플레이어 승
- [ ]  둘다 버스트가 아닌 경우 21에 가까운 이가 승

### 최종 승패 통계 출력

- [ ]  딜러의 총 승패 결과 출력
    - [ ]  출력 형태: `딜러 {winCount}승 {loseCount}패`
- [ ]  각 플레이어의 승패 여부 출력
    - [ ]  출력 형태: `{playerName}: 승` 혹은 `{playerName}: 패`

# Domain

### Suit (enum)

- 스페이드, 하트, 다이아몬드, 클로버를 표현하는 열거형이다.
- 위의 값이 아닌 다른 문자열이 들어오면 예외를 발생시킨다.

### Rank (enum)

- 카드의 숫자 값을 관리하는 열거형이다.
- J, Q, K는 10을 반환한다.
- A는 11을 반환한다.
- 숫자 카드는 해당 숫자 값을 그대로 반환한다.
- 1~10, A, J, Q, K가 아닌 다른 값이 들어오면 예외를 발생시킨다.

### TrumpCard

- Suit와 Rank를 조합하여 하나의 카드 객체를 생성한다.
- 카드의 정보를 확인할 수 있도록 `toString()` 메서드를 제공한다.

### Deck

- 카드 덱을 관리하는 객체이다.
- `List<TrumpCard>` 형태로 카드들을 보관한다.
- 덱을 랜덤하게 생성한다.
- 카드 지급 시 앞에서부터 제거하며 지급한다.
- 덱의 전체 카드 수를 검증한다.
- 카드 중복 여부를 확인한다.

### PlayerName

- 플레이어 이름을 관리하는 값 객체이다.
- 이름의 길이가 10자를 초과하면 예외를 발생시킨다.
- 이름이 공백일 경우 예외를 발생시킨다.
- 이름은 영어(대문자, 소문자) 또는 한글만 허용한다.
- 이름 중간의 공백은 허용한다.
- 두 PlayerName 객체를 비교할 수 있다.

### Player

- 플레이어 정보를 관리한다.
- `name`과 `hand`를 가진다.

### Dealer

- 딜러 정보를 관리한다.
- `hand`를 가진다.

### Players

- 여러 플레이어를 관리하는 객체이다.
- `playerList`를 통해 플레이어 목록을 관리한다.

### Hand

- 플레이어 또는 딜러가 받은 카드 목록을 관리한다.
- 카드를 추가할 수 있다.
- 카드 점수를 계산한다.
- 점수가 21을 초과할 경우 Ace를 1로 변환하여 점수를 조정한다.
- 두 Hand 객체의 점수를 비교할 수 있다.

### Score

- 카드들의 점수를 합산하는 역할을 담당한다.
