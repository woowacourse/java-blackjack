# 블랙잭 미션 저장소

## 페어와 함께한 고민

## 개념(도메인) 고민

### 흑곰

- 사용자
    - 딜러
    - 참여자
- 카드
    - 숫자 / K, Q, J
    - 문양
- Cards (일급 컬렉션)

### 밀란

- 카드
    - 값 (숫자 or 문자)
    - 문양 (스페이드 / 클로버 / 하트 / 다이아)
- 사람
    - 참여자
    - 딜러

---

## 비즈니스 로직 흐름

- 사용자 이름 입력
- 게임 시작
    - 카드 덱 생성
    - 초기 카드 배분
- 게임 중
    - 사용자는 카드를 더 받을지 선택한다.
    - 사용자가 Burst 또는 Blackjack 상태가 되면 다음 참여자로 넘어간다.
    - 모든 참여자의 차례가 끝난 뒤, 한 명이라도 버스트하지 않았다면 딜러가 카드를 받는다.
    - 딜러는 카드 합이 16 이하일 경우 카드를 추가로 받고, 17 이상이면 멈춘다.
- 게임 종료
    - 딜러와 참여자의 승패 결과를 보여준다.

---

## 기능 목록

### 사용자 이름 입력 기능

- [x] 쉼표를 기준으로 이름을 분리한다.
- [x] 5자를 넘는 이름은 허용하지 않는다.

### 게임 시작

- [x] 카드 덱을 생성한다.
- [x] 덱을 셔플한다.
- [x] 모두에게 카드를 한 장씩 나눠준다.

### 게임 중

- [x] 각 참여자는 카드를 받을지 말지 선택한다.
- [x] 스테이 또는 버스트 상태가 되면 다음 참여자로 넘어간다.
- [x] 모든 참여자의 차례가 끝난 뒤 다음 단계로 넘어간다.
- [x] 한 명이라도 버스트하지 않은 참여자가 있으면 딜러는 카드를 받는다.
- [x] 딜러는 카드 합계가 17점 이상일 때까지 카드를 받는다.
- [x] 모든 참여자의 게임 결과를 보여준다.

### 게임 종료

- [x] 딜러가 버스트일 경우
    - 유저가 버스트면 유저 패, 딜러 승
    - 유저가 스테이 상태면 유저 승, 딜러 패

- [x] 딜러가 스테이일 경우
    - 유저가 버스트면 유저 패, 딜러 승
    - 유저가 스테이 상태이면 수를 비교하여 승패를 결정한다.
    - 무승부이면 유저 승, 딜러 패

## 도메인

### 카드(Card)

- [x] 카드의 모양과 카드값으로 하나의 카드를 표현한다.
- [x] 카드 이름을 문자열로 반환한다.

### 카드값(CardValue)

- [x] 카드값의 점수를 관리한다.
- [x] Ace, 숫자 카드, 그림 카드의 점수를 구분한다.

### 모양(Shape)

- [x] 카드의 모양을 관리한다.

### 유저(User)

- [x] 5자를 넘는 이름은 허용하지 않는다.
- [x] 카드를 받을 수 있다.
- [x] 보유한 카드 목록을 관리한다.
- [x] 카드 점수 합을 계산할 수 있다.
- [x] 버스트 여부를 판단할 수 있다.
- [x] 블랙잭 여부를 판단할 수 있다.
- [x] 게임 종료 상태를 판단할 수 있다.

### 딜러(Dealer)

- [x] 딜러 이름은 고정된다.
- [x] 처음 카드 한 장만 공개할 수 있다.
- [x] 카드 합이 16 이하일 경우 카드를 추가로 받는다.
- [x] 카드 합이 17 이상일 경우 멈춘다.

### Cards

- [x] 카드 목록을 관리한다.
- [x] 카드 이름 목록을 반환한다.
- [x] 카드 점수를 계산한다.
- [x] Ace 점수를 상황에 따라 계산한다.
- [x] 블랙잭 여부를 판단할 수 있다.
- [x] 버스트 여부를 판단할 수 있다.

### 덱(Deck)

- [x] 모양과 카드값을 조합하여 하나의 덱을 만든다.
- [x] 덱을 셔플한다.
- [x] 가장 위에 있는 카드를 꺼낸다.

### 게임 결과(GameResult)

- [x] 사용자별 승패 결과를 저장한다.
- [x] 사용자 승리 여부를 조회할 수 있다.
- [x] 유저 승리 수를 계산한다.
- [x] 딜러 승리 수를 계산한다.
- [x] 전체 결과 목록을 순회할 수 있다.

## 추가 기능 구현 목록 - 베팅

### 베팅 금액 입력

- [ ] 각 참여자는 게임 시작 전에 베팅 금액을 입력한다.
- [ ] 참여자별 베팅 금액을 관리한다.

### 블랙잭 규칙 추가

- [ ] 처음 두 장의 합이 21인 경우 블랙잭으로 판단한다.
- [ ] 참여자와 딜러가 모두 블랙잭이면 무승부이다.
- [ ] 참여자만 블랙잭이면 참여자가 승리한다.
- [ ] 딜러만 블랙잭이면 참여자가 패배한다.

### 수익 계산 기능 추가

- [ ] 일반 승리 시 참여자는 베팅 금액만큼 수익을 얻는다.
- [ ] 일반 패배 시 참여자는 베팅 금액만큼 손실을 본다.
- [ ] 무승부일 경우 수익은 0이다.
- [ ] 블랙잭으로 승리할 경우 참여자는 베팅 금액의 1.5배만큼 수익을 얻는다.
- [ ] 참여자가 버스트이면 베팅 금액만큼 손실을 본다.
- [ ] 딜러의 수익은 모든 참여자의 수익을 합산하여 반대로 계산한다.

### 결과 출력 기능 추가

- [ ] 참여자별 최종 수익을 출력한다.
- [ ] 딜러의 최종 수익을 출력한다.
