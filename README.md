# java-blackjack

블랙잭 미션 저장소

# 🃏 구현할 기능 목록

### 1. 입력 처리: 플레이어 이름

- [x]  플레이어 이름을 쉼표를 구분자로 입력받는다.
    - [x]  **검증**: 게임에 참여하는 플레이어 수가 5명 이하인지 검증한다.
    - [x]  **검증**: 플레이어 이름은 1글자 이상 8글자 이하인지 검증한다.
    - [x]  **검증**: 영문자만 포함되었는지 검증한다.
    - [x]  **검증**: 플레이어 이름이 중복되지 않는지 검증한다.

### 2. 카드 덱 생성

- [x]  52장의 카드 덱을 생성한다.
- [x]  카드 덱을 무작위로 shuffle 한다.
- [x]  카드 한 장을 뽑을 수 있다.
- [x]  덱이 비었을 경우 예외 처리한다.

### 3. 카드 분배

- [x]  게임 시작 시 딜러에게 카드 2장을 분배한다.
- [x]  게임 시작 시 플레이어에게 카드 2장을 분배한다.
- [x]  딜러 카드는 1장만 공개한다.
- [x]  플레이어 카드는 모두 공개한다.

### 4. 플레이어 턴 진행

- [x]  플레이어에게 hit / stand 여부를 입력받는다.
    - [x]  **검증**: 입력값이 y 또는 n인지 검증한다.
- [x]  hit 선택 시 카드를 한 장 추가한다.
- [x]  카드를 받을 때마다 플레이어 카드 목록을 출력한다.
- [x]  플레이어의 패가 bust 되면 턴을 종료한다.
- [x]  플레이어가 stand를 선택하면 턴을 종료한다.

### 5. 딜러 턴 진행

- [x]  딜러의 점수가 16 이하이면 hit 한다.
- [x]  딜러의 점수가 17 이상이면 stand 한다.

### 6. 점수 계산

- [x]  카드 점수를 계산한다.
- [x]  Ace를 패 상황에 따라 1 또는 11로 계산한다.
    - [x]  Bust시, Ace를 1로 계산한다.
- [x]  점수가 21을 초과하면 bust로 판단한다.
- [x]  초기 두 장이 21이면 blackjack으로 판단한다.

### 7. 승패 판단

- [x]  플레이어와 딜러의 점수를 비교한다.
- [x]  플레이어 bust 시 딜러의 패와 상관없이 패배 처리한다.
- [x]  딜러 bust 시 플레이어 승리 처리한다.
- [x]  점수가 더 높은 쪽이 승리한다.
- [x]  점수가 같으면 무승부 처리한다.
    - [x]  딜러와 플레이어 모두 21점 일 시, Blackjack 여부에 따라 승패가 처리된다.
    - [x]  딜러와 플레이어 모두 Blackjack이라면, 무승부 처리한다.

### 8. 결과 출력

- [x]  딜러와 플레이어의 모든 카드를 출력한다.
- [x]  딜러와 플레이어의 최종 점수를 출력한다.
- [x]  딜러 기준 승 / 무 / 패 결과를 집계하여 출력한다.
- [x]  모든 플레이어 각각의 승패 결과를 출력한다.

# step2 추가 요구 사항 (배팅 기능 추가)

### 9. 배팅 금액 입력

- [ ]  플레이어는 게임 시작 시 배팅 금액을 입력한다.
    - [ ]  **검증**: 배팅 금액은 숫자만 입력 가능한지 검증한다.
    - [ ]  **검증**: 배팅 금액은 0 이상인지 검증한다.
    - [ ]  **검증**: 배팅 금액은 100,000,000 이하인지 검증한다. (배팅 최대 금액은 1억으로 설정: 너무 큰 금액으로 하는 도박은 위험하기 때문이다.)

### 10. 배팅 결과 계산

- [ ]  게임 결과에 따라 배팅 금액의 수익을 계산한다.
    - [ ]  승리 시 배팅 금액만큼 수익을 얻는다. (1배 수익)
    - [ ]  블랙잭 승리 시 배팅 금액의 1.5배 수익을 얻는다. (1.5배 수익)
    - [ ]  무승부 시 배팅 금액을 돌려받는다. (수익: 0)
    - [ ]  패배 시 배팅 금액을 잃는다. (배팅 금액 만큼 minus)

### 11. 수익 결과 출력

- [ ]  딜러의 총 수익을 계산하여 출력한다. (딜레이어는 플레이어의 수익만큼 돈이 나간다.)
- [ ]  각 플레이어의 수익 결과를 출력한다. 


# 📑 Domain 구조 및 역할 설계
**Card**
- 필드
   - `Rank rank`  - 카드에 따른 점수 보유 (enum)
   - `Suit suit` - 카드 모양 보유 (enum)
- 역할: 카드 식별 

**Hand**
- 필드
   - `List<Card> hand`
- 역할:  점수 합계 계산, 버스트 판단, Ace 1/11 처리

**Deck**
- 필드
   - `List<Card> cards`
- 역할: 게임 시작 시 전체 카드 섞기, 카드 한 장씩 뽑아줌

**Participant**
- `Dealer` 와 `Player`의 부모 클래스
- 필드
  - `Hand hand`
- 모든 참가자는 카드를 받음

**Dealer**
- 역할: 딜러
   - 첫 턴 부여받은 카드 2장 중 1장 비공개
   - 16 이하 hit, 17 이상 stand

**Player**
- 역할: 플레이어
   - 첫 턴 부여받은 카드 2장 모두 공개
   - 사용자 입력에 따라 hit, stand 행동 결정
   - 플레이어 이름 검증

**Players**
- 필드
   - `List<Player> players`
- 역할: 게임 참가 인원 검증, 이름 중복 검증

# 🎯 기능 요구 사항

- 자바 코드 컨벤션을 지키면서 프로그래밍한다.
- 기본적으로 Java Style Guide을 원칙으로 한다.
- indent(인덴트, 들여쓰기) depth를 2를 넘지 않도록 구현한다. 1까지만 허용한다.
- 3항 연산자를 쓰지 않는다.
- 모든 기능을 TDD로 구현해 단위 테스트가 존재해야 한다. 단, UI(System.out, System.in) 로직은 제외
- 핵심 로직을 구현하는 코드와 UI를 담당하는 로직을 구분한다.
- UI 로직을 InputView, ResultView와 같은 클래스를 추가해 분리한다.
- 함수(또는 메서드)의 길이가 10라인을 넘어가지 않도록 구현한다.
- 함수(또는 메소드)가 한 가지 일만 하도록 최대한 작게 만들어라.
- 배열 대신 컬렉션을 사용한다.
- 모든 원시 값과 문자열을 포장한다.
- 줄여 쓰지 않는다(축약 금지).
- 일급 컬렉션을 쓴다. 
- 모든 엔티티를 작게 유지한다. 
- 3개 이상의 인스턴스 변수를 가진 클래스를 쓰지 않는다. 
- 딜러와 플레이어에서 발생하는 중복 코드를 제거해야 한다.
