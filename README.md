# 블랙잭 미션 저장소

## 페어와 함께한 고민

## 개념(도메인) 고민

### 흑곰

* 사용자
    * 딜러
    * 참여자
* 카드
    * 숫자 / K, Q, J
    * 문양
* 카드s (일급 컬렉션)
* 라운드

### 밀란

* 카드
    * 값 (숫자 or 문자)
    * 문양 (스페이드 / 클로버 / 하트 / 다이아)
* 사람
    * 참여자
    * 딜러
* 라운드

## 비지니스 로직 흐름

* 사용자 이름 입력
* 게임 시작
    * 카드 덱 생성
    * 초기 카드 배분
* 게임 중
    * 사용자는 Stay나 Burst 상태일때까지 카드를 한장씩 받는다.
    * 마지막 사용자까지 반복한다.
    *
        1) 만약 모든 사용자가 Burst라면 결과를 보여준다.
    *
        2) Stay가 하나라도 남아있으면 딜러는 카드를 뽑는다.

        * 2-1) 딜러 카드의 합이 16이하일 경우 카드를 추가로 받는다.
        * 2-2) 딜러 카드의 합이 17이상일 경우 멈추고 결과를 보여준다.
* 게임 종료
    * 딜러와 참여자의 승/패를 보여준다.

### 사용자 이름 입력 기능

- [ ] 공백 입력은 허용하지 않는다.
- [ ] 숫자와 특수문자, 끝에 쉼표를 허용하지 않는다.
- [ ] 쉼표를 기준으로 이름을 분리한다.
- [x] 5자를 넘는 이름은 허용하지 않는다.
- [ ] 참여자는 최대 5명이다.

### 게임 시작

- [ ] 카드 덱을 생성한다.
- [ ] 덱을 셔플한다.
- [ ] 모두에게 카드를 한 장씩 나눠준다.
- [ ] 참여자에게만 카드를 한 장씩 추가적으로 나눠준다.

### 게임 중

- [ ] 각 참여자는 카드를 받을지 말지 선택한다.
- [ ] 스테이나 버스트가 나올 경우 다음 참여자로 넘어간다.
- [ ] 만약 다음 참여자가 없을 경우 다음 단계로 넘어간다.
- [ ] 모두 버스트 상태인 경우 결과를 보여준다.
- [ ] 한명이라도 스테이가 있을 경우 딜러는 카드를 받는다.
- [ ] 딜러는 카드 합계가 17점 이상일 때 까지 카드를 받는다.
- [ ] 모든 참여자의 게임 결과를 보여준다.

### 게임 종료

- [ ] 참여자 모두가 버스트일 경우
    -[ ] 딜러가 승리
- [ ] 딜러가 버스트일 경우
    -[ ] 스테이인 참여자가 승리
- [ ] 딜러가 스테이일 경우
    -[ ] 스테이인 참여자들과 딜러를 비교하여 높은 카드 합을 가진 사람이 승리
- [ ] 최종 승패 결과를 보여준다.
