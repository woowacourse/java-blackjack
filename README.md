## 🃏 블랙잭 요구사항

블랙잭 게임은 딜러와 플레이어 중 카드의 합이 21 또는 21에 가장 가까운 숫자를 가지는 쪽이 이기는 게임이다.

- 카드의 숫자 계산은 카드 숫자를 기본으로 한다.
  - 예외로 Ace는 1 또는 11로 계산한다. 
  - King, Queen, Jack은 각각 10으로 계산한다.

- 게임을 시작하면 플레이어는 두 장의 카드를 지급 받으며, 두 장의 카드 숫자를 합쳐 21을 초과하지 않으면서 21에 가깝게 만들면 이긴다. 
  - 21을 넘지 않을 경우 원한다면 얼마든지 카드를 계속 뽑을 수 있다.
- 딜러는 처음에 받은 2장의 합계가 16 이하이면 반드시 1장의 카드를 추가로 받아야 하고, 17점 이상이면 추가로 받을 수 없다.
- 게임을 완료한 후 각 플레이어별로 승패를 출력한다.

### 실행 결과
```text
게임에 참여할 사람의 이름을 입력하세요.(쉼표 기준으로 분리)
pobi,jason

딜러와 pobi, jason에게 2장을 나누었습니다.
딜러카드: 3다이아몬드
pobi카드: 2하트, 8스페이드
jason카드: 7클로버, K스페이드

pobi는 한장의 카드를 더 받겠습니까?(예는 y, 아니오는 n)
y
pobi카드: 2하트, 8스페이드, A클로버
pobi는 한장의 카드를 더 받겠습니까?(예는 y, 아니오는 n)
n
jason는 한장의 카드를 더 받겠습니까?(예는 y, 아니오는 n)
n
jason카드: 7클로버, K스페이드

딜러는 16이하라 한장의 카드를 더 받았습니다.

딜러카드: 3다이아몬드, 9클로버, 8다이아몬드 - 결과: 20
pobi카드: 2하트, 8스페이드, A클로버 - 결과: 21
jason카드: 7클로버, K스페이드 - 결과: 17

## 최종 승패
딜러: 1승 1패
pobi: 승 
jason: 패
```

### 구현 기능 목록

#### 입력
- [x] 참여할 플레이어 이름 요청
  - 입력값이 비어 있을 경우 예외 발생
  - 구분자가 쉼표(,)가 아닐 경우 예외 발생
  - 쉼표, 공백을 제외한 특수 문자가 입력된 경우 예외 발생
  - 중복된 이름이 입력된 경우 예외 발생
  - 딜러 제외 참여 인원이 1명-7명이 아닌 경우 예외 발생
    - 최대 인원 결정 이유: [표준 카지노 룰](https://namu.wiki/w/%EB%B8%94%EB%9E%99%EC%9E%AD(%ED%94%8C%EB%A0%88%EC%9E%89%20%EC%B9%B4%EB%93%9C)#:~:text=%EC%B9%B4%EB%93%9C%EC%9D%98%20%ED%95%A9%EC%9D%B4%2021%EC%A0%90%20%EB%98%90%EB%8A%94%2021%EC%A0%90%EC%97%90%20%EA%B0%80%EC%9E%A5%20%EA%B0%80%EA%B9%9D%EA%B2%8C%20%EB%A7%8C%EB%93%A4%EB%A9%B4%20%EC%8A%B9%EB%A6%AC%ED%95%98%EB%A9%B0%2C%2021%EC%9D%84%20%EC%B4%88%EA%B3%BC%ED%95%98%EB%A9%B4%20%ED%8C%A8%EB%B0%B0%ED%95%9C%EB%8B%A4.%20%EC%82%AC%EC%9A%A9%ED%95%98%EB%8A%94%20%EC%B9%B4%EB%93%9C%EB%8A%94%20%EC%82%AC%EC%A0%81%EC%9D%B8%20%EA%B2%8C%EC%9E%84%EC%97%90%EC%84%9C%EB%8A%94%20%EC%A1%B0%EC%BB%A4%EB%A5%BC%20%EC%A0%9C%EC%99%B8%ED%95%9C%2052%EC%9E%A5%EC%9D%84%2C%20%EC%B9%B4%EC%A7%80%EB%85%B8%EC%97%90%EC%84%9C%EB%8A%94%20%EC%97%AC%EB%9F%AC%20%EB%B2%8C(%EB%B3%B4%ED%86%B5%206%EB%B2%8C)%EC%9D%84%20%EC%82%AC%EC%9A%A9%ED%95%98%EB%A9%B0%2C%20%EC%B0%B8%EA%B0%80%ED%95%98%EB%8A%94%20%EC%9D%B8%EC%9B%90%EC%9D%80%202~8%EB%AA%85%EC%9D%B4%EB%8B%A4.)

- [ ] 카드 더 받을지 여부(`예: y`, `아니오: n`) 요청
  - 입력값이 `y(Y)` 또는 `n(N)`이 아닌 경우 예외 발생

#### 출력
- [ ] 플레이어가 소지한 카드 패 출력
- [ ] 카드 점수 합계 출력
- [ ] 딜러의 카드 드로우 여부 출력
- [ ] 최종 승패 출력

#### 카드 생성
- [x] 총 52장의 카드 생성 후 셔플
  - 문양 : 하트, 스페이드, 클로버, 다이아몬드 4가지  
  - 숫자 : 각 문양 당 2-10, J, Q, K, A 총 13가지

#### 카드 분배
- [x] 카드 2장 분배 경우
  - 시작 시 모든 플레이어에게 분배
- [ ] 카드 1장 분배 경우
  - 딜러를 제외한 플레이어의 카드 총합이 21 미만 + 플레이어의 히트 요청 시 분배
  - 딜러의 카드 총합이 17 미만일 때 분배

#### 카드 점수 계산
- [ ] 플레이어가 소지한 모든 카드의 점수 총합
  - 소지 패에 A가 포함 **된** 경우
    - A를 1 또는 11로 계산
      - A를 1로 계산하는 경우: 하나의 A를 11로 계산하면 버스트가 될 때
    - 21을 초과하지 않는 가장 높은 점수로 계산

#### 버스트 판단
- [ ] 플레이어의 카드 총합이 21을 초과한 경우 버스트(패배)

#### 승패 판단
- [ ] 플레이어 패배 상황
  - [ ] 플레이어가 버스트된 경우
  - [ ] 플레이어의 카드 총합이 딜러보다 낮을 경우
- [ ] 무승부 상황
  - [ ] 딜러와 플레이어의 카드 총합이 같을 경우
- [ ] 플레이어 승리 상황
  - [ ] 플레이어의 카드 총합이 딜러보다 높을 경우
  - [ ] 플레이어는 버스트가 아니지만, 딜러가 버스트된 경우
- [ ] 딜러의 승/패/무 횟수 계산
