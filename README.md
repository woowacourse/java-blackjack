# java-blackjack

블랙잭 미션 저장소

## 사이클1 - 페어 프로그래밍
- 10분으로 정하고 해보고 줄이거나 늘리는 방향으로 수정
- 기능 목록 작성 시에 10분으로 페어프로그래밍 진행
- 코드 작성시에는 15분으로 변경

## 사이클2 - 베팅 기능 추가
- 페어 프로그래밍 결과물에서 베팅 기능을 추가

## 요구사항 
블랙잭 게임을 변형한 프로그램을 구현한다. 블랙잭 게임은 딜러와 플레이어 중 카드의 합이 21 또는 21에 가장 가까운 숫자를 가지는 쪽이 이기는 게임이다.

* 플레이어는 게임을 시작할 때 배팅 금액을 정해야 한다.
* 카드의 숫자 계산은 카드 숫자를 기본으로 하며, 예외로 Ace는 1 또는 11로 계산할 수 있으며, King, Queen, Jack은 각각 10으로 계산한다.
* 게임을 시작하면 플레이어는 두 장의 카드를 지급 받으며, 두 장의 카드 숫자를 합쳐 21을 초과하지 않으면서 21에 가깝게 만들면 이긴다. 21을 넘지 않을 경우 원한다면 얼마든지 카드를 계속 뽑을 수 있다. 단, 카드를 추가로 뽑아 21을 초과할 경우 배팅 금액을 모두 잃게 된다.
* 처음 두 장의 카드 합이 21일 경우 블랙잭이 되면 베팅 금액의 1.5 배를 딜러에게 받는다. 딜러와 플레이어가 모두 동시에 블랙잭인 경우 플레이어는 베팅한 금액을 돌려받는다.
* 딜러는 처음에 받은 2장의 합계가 16이하이면 반드시 1장의 카드를 추가로 받아야 하고, 17점 이상이면 추가로 받을 수 없다. 딜러가 21을 초과하면 그 시점까지 남아 있던 플레이어들은 가지고 있는 패에 상관 없이 승리해 베팅 금액을 받는다.

실행 결과
```
게임에 참여할 사람의 이름을 입력하세요.(쉼표 기준으로 분리)
pobi,jason

pobi의 배팅 금액은?
10000

jason의 배팅 금액은?
20000

딜러와 pobi, jason에게 2장을 나누었습니다.
딜러: 3다이아몬드
pobi카드: 2하트, 8스페이드
jason카드: 7클로버, K스페이드

pobi는 한장의 카드를 더 받겠습니까?(예는 y, 아니오는 n)
y
pobi카드: 2하트, 8스페이드, A클로버
pobi는 한장의 카드를 더 받겠습니까?(예는 y, 아니오는 n)
n
pobi카드: 2하트, 8스페이드, A클로버
jason은 한장의 카드를 더 받겠습니까?(예는 y, 아니오는 n)
n
jason카드: 7클로버, K스페이드

딜러는 16이하라 한장의 카드를 더 받았습니다.

딜러 카드: 3다이아몬드, 9클로버, 8다이아몬드 - 결과: 20
pobi카드: 2하트, 8스페이드, A클로버 - 결과: 21
jason카드: 7클로버, K스페이드 - 결과: 17

## 최종 수익
딜러: 10000
pobi: 10000 
jason: -20000
```

### 게임 참여 사람 이름 입력 기능

게임 참여자를 입력받습니다.
세부 기능 :

* [x] 쉼표 기준으로 분리
* [x] 이름은 2~5자 입력
* [x] 영어 이름만 가능

예외 케이스 :

* 공백이나 빈 문자열의 경우 예외를 반환 해야함.
* 쉼표가 아닌 구분자가 들어있을 경우 예외를 반환 해야함.
* 영어 이름이 아닌 경우 예외를 반환 해야함.
* 2~5자가 아닌 경우 예외를 반환 해야함.

테스트 케이스:

* [x] “”, “ ” → IllegalArgumentException 반환
    * “값을 입력해주세요.” 메시지 출력
* [x] “phobi:jason” → IllegalArgumentException 반환
    * “허용되지 않는 구분자입니다.” 메시지 출력
* [x] “마이찬,도우너” → IllegalArgumentException 반환
    * “영어 이름만 입력해주세요.” 메시지 출력
* [x] “phobii,jasonn” → IllegalArgumentException 반환
    * “이름은 2~5 글자만 가능합니다.” 메시지 출력
* [x] “phobi,jason” → 성공


### 딜러와 참여자 관리 기능

딜러와 참여자가 게임을 진행하는데 카드를 배부 받고 참여자들과 딜러의 게임 승패를 관리한다.
세부 기능:

* [x] 초기 참여자와 딜러 생성 기능

딜러 :

* 기본 2장
* 처음은 한 장만 공개
* 마지막에 16 이하면 한 장 더

참여자 :

* 기본 2장
* 추가 여부 ( y or n )
    * 21이 넘지 않을때까지 계속 반복

### 카드 덱 관리

카드를 생성하고 관리합니다.
세부 기능 :

* 카드 뽑기 기능
    * [x] 카드의 모양을 랜덤으로 반환
    * [x] 카드의 랭크를 랜덤으로 반환
    * [x] 랭덤으로 받은 모양과 랭크를 통해 카드 생성 후 반환
* 카드 중복 검사 기능
    * [x] 기존에 생성된 카드와 중복되는지 검사

### 게임 관리 기능

카드를 분배하고 추가 배부하고 점수를 계산하고 덱의 카드가 버스트 판정이 됐는지 판단한다.
세부 기능:

* 카드 초기 분배 기능
    * [x] 참여자에 대한 정보가 저장되어 있다.
    * [x] 딜러와 참여자에게 2장씩 분배한다.
    * [x] 분배한 카드 2개씩을 출력한다.
    * [x] 이때, 딜러는 2장 중에서 나중에 생성된 카드만 오픈한다.
* 카드 추가 배부 기능
    * [x] 참여자가 추가 배부 여부가 Y일 때, 새로운 카드를 뽑고 해당 참여자에게 배부한다.
    * [x] 단, 현재 카드의 합이 21을 넘어서는 안된다.
* 점수 계산 기능
    * [x] 참여자의 덱 카드들의 값들을 합산한다. (J,Q,K는 10점으로)
* ACE 1/11 판단 기능
    * [x] ACE 카드가 뽑혔을 때, 현재 카드를 가지고 있는 덱의 유리한 방향으로 숫자를 판단한다.
    * (2, A, Q) → A를 1로 판단
      * (2, A, 3) → A를 11로 판단
* 버스트 판단 기능
    * [x] 덱의 카드들의 합산 결과가 21이 초과 한다면 해당 덱은 버스트 상태로 전환한다.

    
### 최종 승패 판정 기능

참여자와 딜러간의 최종 승패를 판정한다.
세부 기능:

* [x] 참여자와 딜러의 결과 계산
* [x] 최종 승패 판정
    * [x] 딜러의 승패를 반환한다.
        * [x] 21을 초과하지 않으면서 21에 가깝게 만드는 자가 승리한다.
        * [x] 합계가 동점일 경우나 모두 버스트일 경우 무승부로 판정한다.
    * [x] 각 참여자 별 게임 결과를 반환한다.

### 베팅 금액 입력 받기

플레이어 별 베팅 금액을 입력 받습니다.

- [X]  플레이어 별 베팅 금액 입력
- [X]  숫자만 입력 가능

### 초기 블랙잭인지 확인

처음 받은 2장의 카드의 합이 21(블랙잭)인지 확인합니다.

- [X]  처음 받은 2장이 21인지 확인
- [X]  블랙잭이라면 상태값 변경

## 최종 수익 계산

최종 수익을 계산합니다.

1. 플레이어 2장 블랙잭, 딜러 2장 블랙잭 = 배팅금액 반환
2. 플레이어 2장 블랙잭 = 1.5배
3. 플레이어 버스트 = -배팅금액
4. 플레이어 승 = 1배
