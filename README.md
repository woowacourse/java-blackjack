# 블랙잭

## 구현할 기능 목록

### 카드
- [x] 카드는 무늬(Suit)와 숫자(Rank)를 가진다
- [x] 무늬는 스페이드, 하트, 다이아몬드, 클로버 4가지이다
- [x] 숫자는 A, 2~10, J, Q, K이며 A는 1 또는 11로 계산한다

### 덱
- [x] 52장의 카드로 구성된다
- [x] 게임 시작 시 카드를 섞는다
- [x] 카드를 한 장씩 뽑을 수 있다

### 베팅
- [x] 플레이어는 게임 시작 전 배팅 금액을 입력한다
- [x] 배팅 금액은 양수여야 한다
- [x] 수익(Profit)은 음수, 0, 양수 모두 가능하다

### 참여자
- [x] 딜러와 플레이어는 카드를 받을 수 있다
- [x] 딜러와 플레이어는 보유한 카드의 점수를 계산할 수 있다
- [x] 플레이어는 히트 상태이고 버스트가 아니면 카드를 더 받을 수 있다
- [x] 딜러는 16 이하이면 카드를 더 받아야 한다
- [x] ACE를 포함할 때 버스트가 나지 않으면 11로, 아니면 1로 계산한다
- [x] 초기 2장으로 합계가 21이면 블랙잭이다

### 게임 진행
- [x] 게임 시작 시 딜러와 플레이어에게 각 2장씩 나눠준다
- [x] 초기 딜 시 딜러는 카드 1장만 공개한다
- [x] 플레이어는 순서대로 히트(y) 또는 스테이(n)를 선택한다
- [x] 플레이어가 히트 시 카드를 받고 현재 카드를 출력한다
- [x] 플레이어가 처음부터 스테이 시 초기 카드를 출력한다
- [x] 플레이어가 히트 후 스테이 시 카드를 출력하지 않는다
- [x] 모든 플레이어 턴 종료 후 딜러 턴을 진행한다

### 결과 계산
- [x] 플레이어만 블랙잭이면 배팅 금액의 1.5배를 수익으로 받는다
- [x] 플레이어와 딜러 모두 블랙잭이면 배팅 금액을 돌려받는다 (무승부)
- [x] 플레이어가 버스트이면 배팅 금액을 잃는다
- [x] 딜러가 버스트이면 남은 플레이어는 배팅 금액을 받는다
- [x] 플레이어 점수가 딜러보다 높으면 배팅 금액을 받는다
- [x] 플레이어 점수가 딜러보다 낮으면 배팅 금액을 잃는다
- [x] 점수가 같으면 배팅 금액을 돌려받는다 (무승부)
- [x] 딜러 수익은 플레이어 수익 합산의 반전값이다

### 입출력
- [x] 참여자 이름을 쉼표 기준으로 입력받는다
- [x] 플레이어별 배팅 금액을 입력받는다
- [x] 배팅 금액에 숫자가 아닌 값 입력 시 예외를 발생시킨다
- [x] 히트 여부를 y/n으로 입력받는다
- [x] y/n 외의 값 입력 시 예외를 발생시킨다
- [x] 초기 딜 결과를 출력한다
- [x] 최종 카드 및 점수를 출력한다
- [x] 최종 수익을 출력한다

## 미션 중 기록

### 기능 추가로 인해 수정한 위치 개수
총 8개 파일 수정, 2개 파일 신규 추가

| 파일 | 변경 유형 | 이유 |
|------|-----------|------|
| `BettingMoney` | 수정 | `final`, 양수 검증, `getAmount()` 추가 |
| `Hand` | 수정 | `isBlackjack()` 추가 |
| `Participant` | 수정 | `BettingMoney` 필드 제거, `isBlackjack()` 위임 추가 |
| `Player` | 수정 | `BettingMoney` 필드 이동, `hasAdditionalCard()` 추가 |
| `Dealer` | 수정 | `super()` 인자 변경 |
| `GameResult` | 수정 | `BLACKJACK` 추가, 수익 계산 책임 부여 |
| `GameResults` | 수정 | 승패 집계 → 수익 계산 구조로 전환 |
| `OutputView` | 수정 | 수익 숫자 출력으로 변경 |
| `Profit` | 신규 | 수익 원시값 포장 |
| `InputView` | 수정 | 숫자 파싱 예외 처리 추가 |

### 사이클1 때보다 수정 범위가 줄었는가 / 늘었는가
**늘었다.**
블랙잭 결과 타입(`BLACKJACK`) 추가로 `GameResult` → `GameResults` → `OutputView` 순으로 연쇄 수정이 발생했다.
수익 계산 구조 자체가 사이클1에 없던 개념이라 수정 범위 증가는 불가피했다.

### 규칙 적용으로 변경한 코드 1곳
**`Participant`에서 `BettingMoney` 필드 제거**
"3개 이상의 인스턴스 변수를 가진 클래스를 쓰지 않는다" 규칙 적용.
`Participant`가 `name`, `hand`, `bettingMoney` 3개를 가지고 있어 위반 상태였고,
`BettingMoney`는 `Player` 고유 개념이므로 `Player`로 이동했다.

### 테스트가 설계를 도운 순간 1회
`GameResult.of()` 시그니처 설계 시, `of(Score, Score)`로 테스트를 먼저 작성하려 했으나
블랙잭 판정에 카드 수 정보가 필요하다는 것을 테스트 케이스 작성 과정에서 발견했다.
`Score`만으로는 "초기 2장 합계 21"과 "3장 이상 합계 21"을 구분할 수 없어
`of(Player, Dealer)` 시그니처로 변경하게 되었다. 테스트가 설계 오류를 사전에 잡아냈다.
